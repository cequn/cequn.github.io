Hexo建站

写在前面的话

心血来潮，想搭建个人博客用于记录平时工作学习中包括健身等遇到的问题以及解决方法，这样不至于在下次碰到时还会花大量时间去找解决途径，同时也可以给身边的朋友一个参考。

开源的博客平台多如牛毛，而且不乏优秀之作，如 Hexo、Jekyll、Wordpress。因为之前的一个同事使用的Hexo的landscape主题，觉得挺喜欢，于是也就选择了Hexo来作为自己的博客平台。

最初的想法是想通过一台Ubuntu 16.04的阿里云服务器来部署Hexo，但因为域名暂时无法认证，也就作罢。转而使用Github Pages（因为免费的）。文章后面会顺便介绍下如何在云服务器上来部署。

GitHub Pages配置

创建Github Pages比较简单，首先你得有这个同性交友网站github的账号，然后去创建一个仓库就可以了，但是这个仓库是有规则的，其格式必须为：

Node.js

curl https://raw.github.com/creationix/nvm/master/install.sh | sh
nvm install stable

Git

sudo apt-get install git

npm install hexo-cli hexo-server -g

hexo init ~/hexo_blog

URL

If your site is put in a subdirectory, set url as ‘http://yoursite.com/child‘ and root as ‘/child/‘

url: http://your_username.github.io #填写你的服务器域名
root: /
permalink: :year/:month/:day/:title/
permalink_defaults: en
…
…

Writing

new_post_name: :title.md # File name of new posts
default_layout: draft #表示每篇博文默认都是草稿，必须经过发布之后才能在博客站点上访问。原来的值是 post
titlecase: false # Transform title into titlecase
…
…

Deployment

Docs: https://hexo.io/docs/deployment.html

deploy:
type: git #通过 Git 将 Hexo 生成的静态内容推送到服务器。
repo: git@github.com:your_username/your_username.github.io.git #
branch: master

npm install hexo-deployer-git –save

hexo generate && hexo deployer

可以使用hexo server先本地预览

npm install hexo-admin –save

CNAME @ 默认 your_username.github.io
CNAME www 默认 your_username.github.io

sudo ln -s /etc/nginx/sites-available/default /etc/nginx/sites-enabled/
```
