open edx platform -- Ubuntu 16.04 native installaton

Contents

  2. 2. 安装edx

最近在折腾edx平台，这个平台比较复杂，加上自己个人技术能力有限，瞎捣鼓了很长一段时间,现记录下过程中的一些使用及问题，方便以后查询。

创建云服务器实例

这里选用的是aliyun的美国服务器，因为在国内，受到防火墙的原因，安装基本会被各种奇怪的问题中断，尝试过很多遍都不成功，换了节点国外节点整个安装过程大概在2-3个小时。
  • 为了节省花费，可选用弗吉尼亚的抢占式实例（比较了下应该是几个区域里最便宜的），个人选择了通用型（2vCPU,8GB,高效云盘50G），总体花费在0.09/时外加0.5/GB的出网流量，停机状态下，话费约为0.03/时，所以这个价格相对来说还是很划算的。

  • 默认的是root登录，可创建个管理员账户

    接下来进行更新

  • 配置~/.ssh/conf添加Host,然后执行ssh-copy-id <Host>将key复制到服务器上，这样可以通过

    整个安装过程约持续2-3个小时，这时候你可以去干点其他事情了 第三步的密码生成是可选的，如果已有了安装的平台，可以将密码拷贝过来，另外如果担心ssh中断问题，可以使用nohup来执行任务。最后，可以检查log目录下的文件来查看安装过否出现错误（目前来看，只要不是国内的节点，安装十分流畅，没有任何错误）

    配置

    大部分情况下所需要修改的配置主要是四个json文件,所在的位置为/edx/app/edxapp,可以发现lms和cms的很多内容还是重复的，这样修改比较繁琐，一个比较好的方法是通过在/edx/app/edx_ansible目录下创建server-vars.yml文件配置变量，可以参考官方的一个例子或者这个， 然后通过以下来更新，这样会覆盖点原来的json文件

    但需要注意的是在执行update之前，先要执行sudo /edx/bin/supervisorctl stop all,停掉lms，cms的服务

    其他

    平台会在根目录下创建一个/edx，主要有四个部分
  • /edx/app 所有的应用模块所在的位置
  • /edx/bin pip,ansible,一些管理命令所在的位置
  • /edx/etc 配置文件的位置

  • /edx/var 数据文件，log查看
